# Commander Token and Tracker Print Manifest

This is the first useful pass for printing extra table aids for the Commander
decks Kyle and his partner are using. The goal is not to build an asset system
yet. The goal is to identify the decks, list the token and helper needs, and
choose the clean next production lane.

## One-lane decision

Build a private, clearly unofficial print-aid pack for casual games:

1. Make a text-first printable kit for the tokens and trackers below.
2. Mark every custom aid `UNOFFICIAL TABLE AID - NOT FOR SALE`.
3. Do not commit downloaded official token/card art into this repo.
4. Use official image galleries or Scryfall only as reference/source lookup,
   unless Kyle explicitly chooses a private official-image print lane.

## Asset routes

### Route A: fastest private-reference print

Use the official Wizards decklist/card-image-gallery pages to find token images,
or use Scryfall search/API to find the matching token names. Print for casual
home play only. Do not commit downloaded official images, redistribute them,
sell them, or remove legal notices.

Use this route when the goal is: "we need exact readable token text tonight."

### Route B: safer custom table-aid pack

Create original, text-first token/tracker cards in Homebrew Forge:

```text
sets/PRINTAIDS/
  sets.csv
  cards.csv
  card_faces.csv
  art_manifest.csv
  export_profiles.csv
assets/packs/table-aids/
  manifest.yaml
docs/
  20_commander_token_print_manifest.md
```

Use simple icons, readable names, color bands, and big rules text. Use generated
or personally licensed art only if art materially helps. This is the best next
implementation lane because it avoids turning the task into asset hunting.

Use this route when the goal is: "we want a reusable pack we can modify and
reprint whenever our decks change."

### Route C: buy real token singles, print trackers only

For common official tokens, buying singles or bulk token lots may be cheaper
than ink and cutting time. Still print custom trackers because official products
rarely include enough of those.

Use this route when the goal is: "we want nice official token art, but do not
want a legal/asset-management project."
